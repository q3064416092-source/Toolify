# REQUIREAI (AI 开发准则)

> **重要提示**：此文件是所有参与本项目开发的 AI 智能体的最高行动准则。
> **核心指令**：任何代码变更必须伴随文档的同步更新。

## 🔴 核心原则：文档即状态

由于 AI 会话可能是断续的，为了保证项目上下文的连续性，你**必须**在每次任务结束前，检查并更新以下两个核心文件。这与编写代码本身同等重要。

### 1. 维护 `0_AGENT_STATE.md` (项目快照)

**何时更新？**

- 当你添加了新的核心组件或文件时。
- 当你修改了数据流向（如：从 Mock 数据改为 API 调用）时。
- 当你改变了核心交互逻辑（如：编辑器从 textarea 改为 contentEditable）时。

**如何更新？**

- **不要**简单追加。
- **必须**阅读旧内容，删除过时的描述，用最新的逻辑替换它。
- 确保 "Last Updated" 字段是最新的日期。
- 保持 "核心模块与逻辑" 部分与代码库的实际行为 100% 一致。

### 2. 维护 `0_TODO.md ` (任务进度)

**何时更新？**

- 当你完成了一个功能点时（将 `[ ]` 改为 `[x]`）。
- 当你在开发过程中发现了新的 Bug 或需求时（添加到待办列表）。
- 当你觉得下一步计划需要调整时。

**如何更新？**

- 保持诚实：如果功能只完成了一半，不要打勾，或者拆分为子任务。
- 保持清晰：为新增的 TODO 项添加简短的说明，必要时指明优先级。

**请记住：你的文档质量决定了下一任 AI 接手时的效率。不做文档的开发是不负责任的。**
